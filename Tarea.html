Definición

El protocolo de transferencia de hipertexto HTTP es un conjunto de reglas que permiten la transmisión de información através de enlaces de hipertexto en la web

En el sitio web de Wikipedia se nos define como el protocolo de comunicación que permite las transferencias de información a través de archivos (XML, HTML…) en la World Wide Web.

Para esto, HTTP utiliza una arquitectura cliente-servidor en el que la información se desplaza de un lado a otro de la cadena de comunicación en un ciclo de solicitud y respuesta, donde el cliente (o agente de usuario) reliza una solicitud al servidor web y este le responde (generalmente de manera afirmativa o negativa) atendiendo a la solicitud que el cliente requirió.


HTTP es un protocolo sin estado, por lo que no guarda ninguna información sobre conexiones anteriores. El desarrollo de aplicaciones web necesita frecuentemente mantener estado. Para esto se usan las cookies, que es información que un servidor puede almacenar en el sistema cliente. Esto le permite a las aplicaciones web instituir la noción de sesión, y también permite rastrear usuarios, ya que las cookies pueden guardarse en el cliente por tiempo indeterminado.

Modelo TCP/IP

HTTP es un protocolo que pertenece a la capa de aplicación del modelo TCP/IP (y también del Modelo OSI) junto a otros protocolos pertenecientes a la misma capa de abstracción que permiten la transmisión de información através de una red, como lo son:

  • FTP
  • SMTP
  • DHCP
  • SSH
  • DNS
    •

y muchos otros...

Fuente Wikipedia: Modelo TCP/IP

Este modelo, además, posee otras múltiples capas de abstracción, las cuales procesan funciones completamente diferentes e independientes unas de otras pero que en conjunto tienen la capacidad de enviar y recibir paquetes de datos entre dos o más terminales de una red local o global.
